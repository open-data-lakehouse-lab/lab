# Contributing to Open Data Lakehouse Lab

First off, thank you for considering contributing to Open Data Lakehouse Lab!

## Our Principles

This project is a personal open-source laboratory. It must remain independent and neutral.

- **No corporate context**: Do not mention any employer, client, company, or internal project.
- **Public data only**: Only use public open datasets.
- **No secrets**: Never include secrets, credentials, or private data.

## How to Contribute

1.  **Search for existing issues**: Before creating a new issue, please check if one already exists.
2.  **Create a new issue**: If you find a bug or have a feature request, please create a new issue.
3.  **Submit a Pull Request**:
    - Fork the repository.
    - Create a new branch for your changes.
    - Make your changes and ensure they follow the project's style and principles.
    - Submit a pull request to the `main` branch.

## Documentation

We use Mermaid for diagrams and Markdown for all documentation.
