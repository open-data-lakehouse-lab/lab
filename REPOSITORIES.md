# Repository Map

The Open Data Lakehouse Lab is organized as a multi-repository project.

## Core Repositories

| Repository | Description |
| ---------- | ----------- |
| `lab` | Central governance, roadmap, architecture, and ADRs. |
| `open-data-lakehouse-lab.github.io` | Project documentation website. |
| `infrastructure` | Infrastructure as Code for local and cloud environments. |
| `datasets-catalog` | Documentation and metadata for public datasets used in the lab. |
| `ingestion` | Data ingestion pipelines and logic. |
| `transformation` | Data transformation logic (dbt, SQL, etc.) for the Lakehouse. |
| `dashboards` | Visualization and dashboard configurations. |
| `api` | Analytics API for data access. |
| `web` | Web portal for data visualization. |
| `azure-local-lab` | Azure-oriented local development environment. |
| `aws-local-lab` | AWS-oriented local development environment. |
| `gcp-local-lab` | GCP-oriented local development environment. |

## Future Repositories

| Repository | Description |
| ---------- | ----------- |
| `orchestration` | Workflow management and scheduling. |
| `quality` | Data quality checks and monitoring. |
| `observability` | Logging, metrics, and tracing for the entire lab. |
